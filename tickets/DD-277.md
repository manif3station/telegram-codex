# DD-277

## Summary

Prevent `telegram-codex.listen` from replaying and auto-replying stale Telegram updates when a session is restarted with imperfect local offset state.

## Method

TDD

## Acceptance Criteria

- `dashboard telegram-codex.listen` skips any returned Telegram update whose `update_id` is older than the next stored offset for that session
- if `listener.offset` is missing but `listener.inbox.jsonl` exists, the listener recovers the next offset from the latest inbox entry instead of replaying old Telegram updates
- repeated starts of the same session do not append or auto-reply the same stale Telegram updates again just because local offset state is imperfect
- Docker tests pass
- coverage reaches `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `telegram-codex` repo
