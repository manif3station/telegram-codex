# DD-273

## Summary

Auto-start the `telegram-codex` listener when Codex launches by installing a `codex` wrapper under `~/.developer-dashboard/cli/` that starts one Telegram listener per Codex session before execing the real Codex binary.

## Method

TDD

## Acceptance Criteria

- `dashboard telegram-codex.install` writes a `codex` launcher wrapper into `~/.developer-dashboard/cli/`
- the wrapper starts one listener per Codex session before execing the real Codex binary
- the wrapper persists listener pid and log files under `~/.telegram-codex/<session-id>/`
- duplicate listener launches for the same session are prevented
- the wrapper honors `TELEGRAM_BOT_TOKEN`, `CODEX_SESSION_ID`, and `TELEGRAM_CODEX_SESSION_ID`
- docs explain that Telegram-first communication depends on starting Codex through the installed wrapper path
- Docker tests pass
- coverage reaches `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `telegram-codex` repo
