## Summary

Add Telegram-native slash command handling for managed `telegram-codex` sessions so supported commands like `/status` are answered directly by the skill instead of being forwarded into Codex as ordinary prompt text, and make `/status` mirror the real live Codex TUI panel when that session is attached.

## Acceptance Criteria

- paired Telegram `/status` requests are handled directly by `telegram-codex` instead of being sent to Codex as ordinary prompt text
- when the paired Codex session is live in a tmux-backed TUI, Telegram `/status` returns the rendered Codex status panel from that live pane
- when no live tmux-backed pane exists for that session, Telegram `/status` returns an explicit unavailable message instead of a synthesized local summary
- Telegram slash command parsing accepts the normal `@botname` suffix form
- unsupported Telegram slash commands are rejected explicitly instead of being forwarded into managed Codex reply generation
- Docker tests pass
- coverage reaches `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `telegram-codex` repo
