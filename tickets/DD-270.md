# DD-270

## Summary

Build the `telegram-codex` skill with a Perl CLI plus a generated local Codex plugin so Telegram messages, photos, files, downloads, and replies can flow through Codex.

## Method

TDD

## Acceptance Criteria

- `dashboard telegram-codex.install <TELEGRAM_BOT_TOKEN>` scaffolds the local Codex plugin bridge under the chosen plugin root and writes the marketplace entry plus plugin `.env`
- `dashboard telegram-codex.get-me` returns the configured bot identity from Telegram Bot API
- `dashboard telegram-codex.updates` polls inbound Telegram updates and returns text, photo, and document metadata
- `dashboard telegram-codex.download <FILE_ID>` downloads a Telegram file locally
- `dashboard telegram-codex.reply <CHAT_ID> <TEXT>` sends a text reply to Telegram
- `dashboard telegram-codex.send-photo <CHAT_ID> <PHOTO_PATH>` sends a local image file to Telegram
- `dashboard telegram-codex.send-document <CHAT_ID> <DOCUMENT_PATH>` sends a local file to Telegram
- `dashboard telegram-codex.auto-reply-start` replies to recent `/start` updates
- the installed plugin includes a stdio MCP bridge that exposes the same Telegram message, download, and reply tools for Codex
- Docker tests pass
- coverage reaches `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `telegram-codex` repo
