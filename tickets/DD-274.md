# DD-274

## Summary

Auto-start the `telegram-codex` listener from the real `codex` command path by provisioning a managed wrapper into a user PATH directory during skill install.

## Method

TDD

## Acceptance Criteria

- `dashboard skills install telegram-codex` provisions a managed `codex` wrapper into a user PATH directory ahead of the npm global Codex binary
- the managed wrapper starts one listener per Codex session before execing the real Codex binary
- the wrapper persists listener pid and log files under `~/.telegram-codex/<session-id>/`
- duplicate listener launches for the same session are prevented
- the wrapper honors `TELEGRAM_BOT_TOKEN`, `CODEX_SESSION_ID`, and `TELEGRAM_CODEX_SESSION_ID`
- docs explain that normal `codex` startup uses the managed wrapper after skill install
- Docker tests pass
- coverage reaches `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `telegram-codex` repo
