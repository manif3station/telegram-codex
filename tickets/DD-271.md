# DD-271

## Summary

Add an always-on listener mode to `telegram-codex` so the bot can long-poll Telegram, persist its offset, log inbound messages locally, and send immediate replies without waiting for a manual operator poll.

## Method

TDD

## Acceptance Criteria

- `dashboard telegram-codex.listen` starts a long-poll listener for the configured bot
- the listener persists the Telegram update offset locally so messages are not reprocessed on restart
- the listener writes inbound message metadata to a local inbox ledger under the skill-managed runtime area
- the listener sends an immediate auto-reply for inbound text, photo, and document messages with configurable reply text
- the listener supports a bounded test mode so the behavior can be verified without an infinite loop
- the installed local Codex plugin documentation describes the always-on listener workflow clearly
- Docker tests pass
- coverage reaches `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `telegram-codex` repo

## Notes

- listener runtime state defaults to `~/.telegram-codex/`
- token discovery must work from the active project `.env`, a parent/root `.env`, the skill `.env`, or the process environment
