# DD-285

## Summary

Tighten the DD collector polling contract so `telegram-codex.start` writes the session-suffixed `dashboard telegram-codex.check-message <session-id>` command, rewrites mismatched collector `cwd` values to the current workspace, prevents the same session-suffixed polling process from running twice, and makes the collector-owned worker resume the persisted `codex.session` target automatically for real Telegram replies.

TDD

## Acceptance Criteria

- `dashboard telegram-codex.start` writes or heals the collector command as `dashboard telegram-codex.check-message <session-id>`
- duplicate collector entries for the same session are removed automatically
- same-name collectors with the wrong `cwd` are rewritten to the current workspace
- a second `telegram-codex.check-message <session-id>` process exits without starting duplicate polling work when one for that same session suffix is already running
- `dashboard telegram-codex.check-message <session-id>` remains the long-running polling loop that receives text and multimedia update metadata and routes Telegram replies back through the persisted active Codex session target
- docs explain the session-suffixed command and no-overlap behavior clearly
- Docker tests pass
- coverage reaches `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `telegram-codex` repo
