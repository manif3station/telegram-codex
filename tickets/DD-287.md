# DD-287

## Summary

Make the managed collector-owned Telegram reply path download inbound media into the session runtime before Codex replies, expose those local files in the Codex prompt, and support outbound managed attachment replies for photos, audio, and documents.

## Acceptance Criteria

- the collector-owned `dashboard telegram-codex.check-message <session-id>` worker downloads inbound supported Telegram media into the session runtime before it asks Codex to reply
- the Codex reply prompt exposes the downloaded local paths and explicitly tells Codex those files are already available locally for inspection
- managed Codex replies can return attachment directives that send photos, audio, and documents back to Telegram instead of only plain text
- the public skill surface includes `dashboard telegram-codex.send-audio`
- docs explain the inbound-download and outbound-attachment reply contract clearly
- Docker tests pass
- coverage reaches `100%`
- `cover_db` is cleaned
- `README.md`, `docs/`, `Changes`, and `tickets/` are updated
- release commit and push complete from the `telegram-codex` repo
